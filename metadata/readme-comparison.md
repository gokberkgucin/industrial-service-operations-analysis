# README Varyant Karşılaştırması

Bu karşılaştırma, README dosyasının repoda bir portal olarak çalışması gerektiği kabulüyle yapıldı. README iki ana yazının yerini almamalı; konuyu, okuma yönünü ve paylaşım sınırlarını hızlıca anlatmalıdır.

| Varyant | Güçlü taraf | Zayıf taraf | Uygunluk |
|---|---|---|---|
| `README-concise.md` | Hızlı okunur, iki ana yazıya net yönlendirir, portal rolünü iyi taşır. | Daha az bağlam verir; akademik çerçeve kısa kalır. | Yüksek |
| `README-detailed.md` | Teknik ve bağlamsal açıklaması daha güçlüdür; repoyu ilk kez gören kişiye daha fazla yön verir. | README'nin ana yazıların yerini alma riski artar. | Orta-yüksek |
| `README-academic.md` | Dili daha kontrollü ve akademiktir; sorumluluk notu nettir. | GitHub portföyü için fazla mesafeli ve kuru kalır. | Orta |

## Seçim

Final README için `README-concise.md` temel alındı.

Gerekçe: Bu repo yapısında asıl içerik iki ana Markdown yazısında duruyor. README'nin görevi okuyucuyu o yazılara taşımak, repodaki sınırları belirtmek ve hızlı bir ilk izlenim vermek. Bu nedenle kısa ama yönlendirici olan varyant en dengeli seçenek.

## Final README'ye taşınan unsurlar

- Repo konusunu açıklayan kısa giriş
- İki ana yazıya relative link
- Teknik ağırlığın ikinci yazıda olduğunu belirten kısa not
- Repo yapısını gösteren Mermaid diyagramı
- Okuma sırası
- Yayın/sorumluluk notu

## Bilerek dışarıda bırakılanlar

- Ana yazıların ayrıntılı özeti
- Gereksiz pazarlama dili
- Kaynak makale sonuçlarını aşırı genelleyen ifadeler
- Kurumsal iç bilgi izlenimi verebilecek detaylar
